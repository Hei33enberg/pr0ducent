# Security

If you believe you’ve found a **security vulnerability**, please **do not** open a public GitHub issue with exploit details.

- Prefer [GitHub private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability) if enabled on this repository.
- Otherwise contact the maintainers with a clear description, affected component, and steps to reproduce.

This project is open source (MIT); deployers are responsible for their own secrets, RLS review, and Supabase/Edge configuration.
